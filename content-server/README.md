# content-server/

Server-only content-release artifacts, generated by `pnpm content:build`:

```
releases/<release-id>/validation.json   structural validation manifest
releases/<release-id>/assessment.json   canonical answers for objective questions
releases/<release-id>/checksums.json    sha256 of learner/validation/assessment
```

**Trust boundary, not a secrecy mechanism.** Similar values also appear in
the public learner release; what matters is that the server derives
correctness and ratings from ITS copy (ADR-006), never from client-submitted
values. This directory must never be served as a static public directory and
must never be imported into browser code.

Generated files — do not edit by hand. The enriched JSON under `data/` plus
the Python scripts remain the sole content-authoring authority (ADR-003).
Regenerate with `pnpm content:build`; releases are immutable and older
release directories are never overwritten by unrelated builds.
