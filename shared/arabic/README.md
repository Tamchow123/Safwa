# shared/arabic

Future home of Arabic text utilities (Phase 3): the comparison normaliser
(NFC + invisible-character stripping + trim, preserving ḥarakāt/shaddah/
hamzah), the TypeScript twin of `scripts/arabic-extract.py`, and the
study-component natural-key builder shared by client and server.

Arabic handling rules are mandatory — see `CLAUDE.md` ("Arabic data-handling
rule" and "Arabic comparison policy").

No implementation exists yet by design (Phase 1 creates boundaries only).
